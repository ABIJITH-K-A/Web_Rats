TNWEBRATS – AUTH & ROLE ACCESS SYSTEM (FINAL)

---

1. OWNER ACCESS SYSTEM

---

RULES:

* Owners CANNOT sign up normally
* Only login using INVITE KEY
* Invite key = one-time use
* Maximum 2 owners allowed in system

---

OWNER LOGIN FLOW

1. Go to: /owner-login

2. Enter:

   * Email
   * Password
   * Invite Key

3. Backend checks:

   IF email (Gmail), name, and password match records
   AND inviteKey is valid AND unused
   AND total owners < 2:

   → Allow account creation/login
   → Mark inviteKey as used
   → Assign role = "owner"
   → Grant highest authority access (can view all profiles, payments, and payrolls across all roles)

   ELSE:
   → Reject access

   NOTE:

   * Super Admins and lower roles can only view data of roles below them in hierarchy

---

INVITE KEY STRUCTURE (Firebase)

Collection: inviteKeys

Fields:

* key: "AJDN-AKSJ-1234-1234"
* role: "owner"
* isUsed: false
* createdAt

---

OWNER LIMIT CHECK

Before creating owner:

* Count users where role == "owner"

IF count >= 2:
→ Block creation

---

2. STAFF AUTH SYSTEM (SEPARATE PORTAL)

---

ROLES:

* super_admin
* admin
* manager
* worker

---

ACCESS PAGE:

/staff-login

(Separate from client login)

---

SIGNUP RULES:

* Only allowed via:
  ✔ Invite key created by a higher role
  ✔ Invite keys (recommended)

---

STAFF LOGIN FLOW:

1. Go to: /staff-login

2. Enter:

   * Email
   * Password

3. System checks:

   * User exists
   * Role is valid (not client)

4. Redirect based on role:

   * super_admin → /superadmin
   * admin → /admin
   * manager → /manager
   * worker → /worker

---

STAFF SIGNUP FLOW (INVITE KEY REQUIRED):

1. Go to: /staff-signup

2. Enter:

   * Name
   * Email
   * Password
   * Invite Key

3. System checks:

   * Invite key exists
   * Invite key is not used
   * Invite key role matches assigned role

4. If valid:

   * Create user with role from invite key
   * Mark invite key as used

5. If invalid:

   * Reject signup

NOTE:

* Invite keys are created by owners first , then other roles except workers can make invite key and can only make them to get lower role invite keys.
* Each invite key is tied to a specific role (admin / manager / worker)
* Signup is NOT allowed without a valid invite key
